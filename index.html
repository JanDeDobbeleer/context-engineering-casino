♣ The House Rules

Context That Persists

Define the rules once — they apply to every hand

🎯
What Instructions Do
  • Set coding style & conventions
  • Define project-specific patterns
  • Specify preferred frameworks
  • Establish documentation standards
  • Share team knowledge with the AI
✨
Why They Matter
  • Consistent AI responses across sessions
  • No need to re-explain context every prompt
  • Team-wide shared understanding
  • Version-controlled with your code
  • Works across multiple AI agents
The Dealer's Edge: Good instructions reduce repeated context and let you delegate tasks with confidence.
♣ Three Decks

Three Ways to Stack the Deck

Instruction file types & their scope

.github/copilot-instructions.md Global
Single file — applies to all chat requests in your workspace automatically. Works in VS Code, Visual Studio, and GitHub.com.
*.instructions.md Conditional
Multiple files with glob patterns — different rules for different file types. e.g., applyTo: "**/*.py"
AGENTS.md Multi-Agent
Universal format for multiple AI agents — Copilot, Claude Code, Cursor. Root or subfolders for different project areas.
Tip: All files combine when sent to the AI. Use copilot-instructions.md for global rules, .instructions.md for language-specific, and AGENTS.md for cross-tool compatibility.

♥ Section Two

Context Engineering

Understanding how AI processes information — and spending your tokens like a professional card counter.

♥ The Discipline
"Context engineering is the delicate art and science of filling the context window with just the right information for the next step."
— Andrej Karpathy, June 2025
🎰
Beyond Prompt Engineering
  • Prompt engineering: narrow focus on wording
  • Context engineering: everything the model sees
  • Instructions, history, tools, retrieved docs, images
🃏
Why It Matters
  • Model quality is fixed — context is in your control
  • Garbage in → garbage out at any model size
  • Right context = dramatically better results
The House Rule: You can't change the deck. But you control exactly which cards go on the table. That's your edge.
♥ Your Stack

The Context Window is Your Stack

Finite, shared, and burning with every turn

📏
The Practical Limit
Determines how much conversation history, documents, or code the model can "see" simultaneously. Everything competes for the same space.
📊
Size Varies by Model
Ranges from a few thousand tokens (early models) to millions (modern models like Claude 4). Bigger stack doesn't mean infinite — it still fills up.
↔️
Input vs. Output
Input windows are far larger than output limits. Models can read a 500-page book but won't recite it back — and reading it costs chips.
🚫
Beyond the Window
Information outside the context window is forgotten — like cards burned from the deck. The model can't reference them at all.
⚠️ 200K tokens ≈ 500-page book. Sounds huge — until every turn burns another hand.

♥ Counting Your Outs

Managing Your Stack

Context compression strategies

🗜️
Summarize the History
Ask AI to summarize the conversation. Start a new hand with the summary as your opening bet. Retains key decisions without the full history.
✂️
Selective Context
Only attach files relevant to the current task. Use #file and @workspace selectively. Remove what you no longer need.
⚡
Prompt Caching
Cache static portions — system prompts, large docs. Reduces cost and latency. The house charges less when you cache. Supported by Anthropic and OpenAI.
🔄
Agentic Compacting
/compact in Copilot CLI. Claude Code auto-compacts at 95% context usage. Let the dealer shuffle for you when the deck runs thin.

♠ The Agent's Profile

Configuring Your Dealer

name: code-reviewer
description: Reviews code quality
tools: Read, Grep, Glob
model: haiku
permissionMode: plan
color: blue
---
You are a code reviewer...
🔧 Tools & Permissions
  • tools — allowlist (inherits all if omitted)
  • disallowedTools — denylist
  • permissionMode — default / acceptEdits / auto / plan
⚙️ Behaviour
  • model — sonnet / opus / haiku / inherit
  • maxTurns — cap on agentic turns
  • isolation — worktree for git isolation
  • memory — user / project / local
